# 请假系统 (H5版) 细化需求文档

## 1. 系统概述
这是一个为移动端设计的H5请假管理演示系统。系统采用纯前端技术实现，数据通过本地JSON文件初始化，并利用浏览器LocalStorage进行模拟持久化。所有写入操作将通过UI提示反馈。

## 2. 角色与权限
### 2.1 普通员工 (Staff)
- **个人工作台**: 查看各类假期的剩余天数余额。
- **请假申请**: 
    - 自动关联姓名、入职日期、申请日期。
    - 选择请假类型（年假、病假等）、起止时间、原因、附件。
    - 校验：请假时长不可超过余额。
- **请假记录**: 查询本人历史记录，支持状态过滤（待审批、已通过、已拒绝）。
- **流程通知**: 在工作台模拟查看审批结果通知。

### 2.2 直线经理 (Manager)
- 具备普通员工所有功能。
- **审批中心**: 审批下属提交的请假申请（一级审批）。

### 2.3 人事 (HR)
- 具备所有员工功能。
- **二级审批**: 在直线经理通过后，进行最终的人事审批。
- **员工管理**: 维护全公司员工信息（入职/离职状态、部门、直线经理）。
- **假期配置**: 维护假期类型（年假、病假、产假等）及规则（是否每年清零、是否强制附件）。
- **额度管理**: 调整特定员工的年度假期额度。
- **数据导出**: 查询并导出（模拟）全公司请假报表。

## 3. 核心流程
### 3.1 请假流程
1. 员工提交 -> 状态变为“待经理审批”。
2. 直线经理审批 -> 通过后变为“待人事审批”，拒绝后返还额度。
3. 人事审批 -> 通过后流程结束，余额已扣除。

### 3.2 数据处理
- **初始化**: 页面加载时从 `data/*.json` 读取初始数据。
- **模拟写入**: 点击“提交/保存”时，弹出提示框“操作成功（演示环境）”，并同步更新一次 LocalStorage。
- **本地化**: 所有的库（如图标库、样式库）均存放在本地 `assets/lib/` 目录下。

## 4. UI/UX 设计要求
- **肤色**: 采用现代深/浅色模式切换，磨砂玻璃质感。
- **动画**: 页面切换使用平滑过渡动画。
- **移动端优化**: 适配刘海屏，大点击区域，下拉刷新模拟。

## 5. 技术栈清单
- 核心：HTML5, CSS3 (Variables, Flex, Grid), Vanilla JS (ES6)。
- 图标：Lucide Icons (本地 SVG)。
- 字体：本地部署字体文件。
- 数据：本地静态 JSON + JS 数据控制器。
